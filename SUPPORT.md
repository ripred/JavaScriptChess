# Support

Use the repository documentation first:

- [README](README.md)

Open a GitHub issue when you have a reproducible bug, documentation problem, or feature request.

For support requests, include:

- Project version, release, or commit.
- Operating system, runtime, compiler, or tool versions.
- The command, input, workflow, or example involved.
- A small reproduction that demonstrates the problem.
- Exact error output, logs, screenshots, or observed behavior.

General debugging is easier to help with when the report includes the smallest command or input that still shows the issue.
